# نظام استمارة تسجيل الطلاب

---

## 1. التقنيات المستخدمة

* **Frontend:** Next.js
* **Backend:** Laravel API (مع استخدام Laravel Sanctum للمصادقة وحماية الـ APIs)
* **OTP Service:** خدمة خارجية للتحقق عبر رمز OTP

---

## 2. الصفحة الرئيسية (Landing Page)

* تصميم حديث ومتجاوب (Responsive) يعمل على جميع الأجهزة (موبايل، تابلت، كمبيوتر).
* عرض عداد تنازلي (Countdown Timer) يوضح الوقت المتبقي لفتح التسجيل.
* عند انتهاء العداد أو عند تفعيل التسجيل من لوحة التحكم، يظهر زر **"سجل الآن"**.

---

## 3. استمارة التسجيل

* **حفظ تلقائي للبيانات أثناء التسجيل:** إذا أغلق المستخدم الصفحة بالخطأ، يمكنه المتابعة لاحقاً دون إعادة تعبئة جميع الحقول من جديد.
* **شريط تقدم (Progress Bar) لتحسين تجربة المستخدم:**
  * الخطوة الأولى: البيانات الشخصية.
  * الخطوة الثانية: التحقق من الهاتف.
  * الخطوة الثالثة: تأكيد التسجيل (صفحة النجاح).

### 3.1 البيانات الإلزامية

1. الاسم الثلاثي للطالب.
2. رقم الهاتف.
3. عنوان السكن.
4. الجنس.

### 3.2 بيانات إضافية

**هل أنت منتمي إلى نقابة المهندسين؟**

* نعم.
* لا.

#### في حالة اختيار "نعم"

* يظهر حقل إلزامي لرفع صورة هوية نقابة المهندسين.

#### في حالة اختيار "لا"

* يتم الانتقال مباشرة إلى الخطوة التالية.

### 3.3 التحقق عبر واتساب

* إدخال رقم هاتف مسجل على واتساب.
* إرسال رمز تحقق (OTP) إلى الرقم.
* إكمال التسجيل بعد إدخال الرمز الصحيح وتأكيده.
* **إعدادات OTP:**
  * تحديد مدة صلاحية الكود (مثلاً 5 دقائق).
  * منع طلب أكثر من عدد معين من الأكواد خلال فترة قصيرة (حد أقصى 5 محاولات OTP خلال 15 دقيقة).
  * إعادة إرسال الكود بعد 25 ثانية.

---

## 4. صفحة نجاح التسجيل (Success Page)

بعد إكمال التسجيل:
* عرض رسالة نجاح للمستخدم.
* إعطاء رقم طلب تلقائي وفريد.
* إمكانية تنزيل أو طباعة إشعار التسجيل.

---

## 5. لوحة التحكم (Admin Panel)

### 5.1 إدارة التسجيل

* فتح أو إغلاق استمارة التسجيل يدوياً.
* تحديد تاريخ ووقت فتح التسجيل.
* تحديد تاريخ ووقت إغلاق التسجيل.
* إمكانية تفعيل التسجيل فوراً (Open Now).
* تعديل عنوان صفحة التسجيل.
* تعديل النصوص والتعليمات الظاهرة على الصفحة.
* تغيير موعد العد التنازلي دون الحاجة لمبرمج.

### 5.2 إدارة الطلبات

* عرض جميع الطلبات المسجلة في جدول.
* عرض وقت وتاريخ التسجيل لكل طالب.
* عرض جميع بيانات الطالب المسجلة.
* عرض صورة هوية نقابة المهندسين في حال تم رفعها.
* توضيح حالة العضوية في نقابة المهندسين (نعم / لا).
* **حالة الطلب** (تظهر في لوحة التحكم فقط)، وتشمل:
  * جديد.
  * قيد المراجعة.
  * مقبول.
  * مرفوض.
* **البحث والتصفية المتقدمة** داخل قائمة المسجلين بواسطة:
  * الاسم.
  * الهاتف.
  * حالة الطلب.
  * التاريخ.
  * المنتمي للنقابة (نعم / لا).
* **إمكانية التصدير:**
  * الصيغ المدعومة: Excel, CSV, PDF.
  * إمكانية تصدير النتائج المفلترة فقط (بناءً على البحث والتصفية الحاليين).

### 5.3 نظام التحكم بالميزات (Feature Flags)

من لوحة التحكم يمكن تفعيل أو تعطيل الميزات التالية ديناميكياً:
* تفعيل / تعطيل نظام التحقق عبر رمز OTP.
* تفعيل / تعطيل خيار رفع الهوية (لنقابة المهندسين).

### 5.4 لوحة الإحصائيات

* عدد المسجلين الكلي.
* عدد المنتمين إلى نقابة المهندسين.
* عدد غير المنتمين.
* عدد التسجيلات اليومية.

## 6. هيكل قاعدة البيانات (Database Schema)

يتضمن النظام الجداول الأساسية التالية:

| الجدول | الوصف |
|---|---|
| `users` | لإدارة حسابات المستخدمين الأساسية. |
| `registrations` | لتخزين طلبات تسجيل الطلاب وتفاصيل الاستمارة (الاسم، الهاتف، العنوان، الجنس، حالة الطلب). |
| `otp_verifications` | لتخزين عمليات التحقق ورموز OTP المرسلة ومدى صلاحيتها ومحاولات التحقق. |
| `attachments` | لإدارة وحفظ مسارات الملفات المرفوعة (صور الهوية) المرتبطة بالتسجيلات. |
| `settings` | لحفظ إعدادات النظام ولوحة التحكم (نصوص الاستمارة، التوقيت، Feature Flags). |
| `activity_logs` | لتسجيل حركات ونشاطات المشرفين والعمليات التي تتم داخل لوحة التحكم. |
| `admins` | لإدارة حسابات مسؤولي النظام المخولين بالدخول للوحة التحكم. |
| `roles` | لتحديد الأدوار والمسميات الوظيفية للمشرفين (مدير عام، مراجع طلبات، مشرف). |
| `permissions` | لتحديد الصلاحيات المتاحة لكل دور (التصدير، تعديل الحالات، التحكم بالميزات). |

## 7. متطلبات إضافية

### 7.1 الأمان والحماية

* **حماية من الـ Spam:**
  * حد أقصى للتسجيل: 3 تسجيلات فقط من نفس الـ IP خلال ساعة واحدة.
* تخزين الصور بشكل آمن (باستخدام Storage منفصل ومستقل للصور).
* **التحقق من صحة البيانات المدخلة:**
  * التأكد من صحة رقم الهاتف العراقي.
  * منع الأحرف غير المناسبة في الاسم.
  * منع الأرقام داخل الاسم.

### 7.2 التقنيات والبنية

* تسجيل وقت إنشاء الطلب بشكل تلقائي.
* دعم اللغة العربية بشكل كامل واتجاه RTL.
* **إصدارات الواجهة البرمجية (API Versioning):** البدء من `/api/v1` لضمان عدم تعطل التطبيقات أو التكاملات القديمة عند تطوير نسخة جديدة مستقبلاً.

### 7.3 مراقبة النظام وتتبع الأداء (System Monitoring)

* استخدام **Laravel Telescope** في بيئة التطوير لتتبع الطلبات والـ DB Queries.
* استخدام **Laravel Horizon** لمراقبة وإدارة الطوابير والمهام الخلفية (Queues / Redis).
* استخدام **Sentry** لتتبع وتسجيل الأخطاء البرمجية تلقائياً في بيئة الإنتاج (Production).
